## Substack 長版

### 我花了幾週，搞懂了 robot learning 的一條完整路徑——而且大部分沒花到錢

#### 起點：一個看似多餘的問題

現在的機器人有種叫 **VLA（Vision-Language-Action）** 的模型，你對它說「把湯罐收進籃子」，它看著畫面就能直接動手。既然 VLA 已經聽得懂人話，**為什麼還需要在它上面再疊一層 LLM？**

因為 VLA 一次只做得了「一個動作」。「把兩樣東西**都**收好」「擦桌子（它不會）」「把**那個**收起來（哪個？）」——這種**拆解、排序、判斷自己能力邊界、聽不懂就反問**的事，VLA 做不到。這是「大腦」的活，不是「小腦」的活。

我想親手把這層大腦做出來，並證明它跟底層的手能力**完全解耦**。

#### 第一階段：純模擬，把「大腦」做出來

我用 Claude 當編排大腦（L3），底下接真的機器人模擬器（LIBERO）+ 官方 VLA（SmolVLA）。然後我刻意去測它的**邊界**，而不是只拍成功畫面：

- 「都收好」→ 它自動拆成兩步、依序執行 ✅
- 「幫我擦桌子」→ 它判斷超出能力，**直接拒絕、不亂動** ⛔
- 「把那個收起來」→ 它**反問**我是哪個，而不是瞎猜 ❓
- 長程任務（官方模型實測只有 ~5% 成功率）→ 它真的失敗了 3 次，然後**誠實放棄並講明原因**，而不是假裝成功 🛑

最後一點我特別在意：**把「失敗」當成一等公民**。跑了 4 輪、15 次，結局一致是誠實中止——這個救回邏輯靠單元測試保證，不靠僥倖拍到一支好看的影片。

#### 第二階段：零硬體，跑通真機器人的學習鏈

純模擬證明了大腦，但我沒碰過「真機器人資料怎麼變成模型」這條鏈。問題是——我不想為了寫學習心得就去買一支機器手臂。

結果發現：**根本不用買**。社群（Hugging Face）上有大量別人用真 SO-101 手臂遙控採集的資料集。我拿其中一個（50 段真實抓放示範），在免費的 Kaggle GPU 上，把官方預訓練的 VLA 重新微調（fine-tune）。

訓練的 loss 從 0.41 一路降到 0.14，模型確實在「模仿」那支真手臂的動作。最後我把自己訓練出來的模型公開上傳（[Kaminoikari/smolvla-so101-pickplace-ft](https://huggingface.co/Kaminoikari/smolvla-so101-pickplace-ft)）。**整條「讀真資料 → 微調 → 產出模型」的鏈，我親手走完了，花費接近零。**

#### 第三階段：做工程，而不是只跑通

我還做了一件「無聊但關鍵」的事：原本每次讓模型動作，系統都要重新載入一次模型，慢得要命（每個任務約 4.5 分鐘）。我把它改成「模型只載一次、常駐重用」。

但我沒有自己說「變快了」就算數——我寫了一支**對照工具**，讓新舊兩種做法跑同一批任務，**確認兩邊的成功/失敗結果完全一致**（加速不能改壞正確性），再量加速幅度：穩態約 **2 倍快**、每個任務省下約 139 秒。

而且這趟驗證**抓到一個只有在真 GPU 上才會現形的 bug**——本機那 57 個測試全綠都測不出來。這提醒我：mock 測試證明邏輯，但有些東西只有真環境會告訴你。

#### 誠實的界線

我**還沒做到**的是：把模型接上一支實體手臂，量真實世界的成功率。我全程是純模擬 + 真資料，沒有真硬體的閉環。這條界線我清楚標著，不假裝跨過去了。

但我走過的這條路——**編排大腦 → 真資料微調 VLA → 工程化加速並驗證**——正是這個領域真正在做的事。下一步要不要買那支手臂，我現在是站在「已經會整條 pipeline」的位置上決定，而不是盲買。
